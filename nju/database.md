## 第二章

### 数据库各部分命名

| 数据库 | 关系 | 文件系统 |
| ------ | ---- | -------- |
| 表     | 关系 | 记录文件 |
| 列     | 属性 | 域       |
| 行     | 元祖 | 记录     |
| 表头   | 模式 | 记录类型 |

1. 数据库是表的集合，表的标题是表的列名的集合，表的标题也被称作关系模式；

2. 属性的域代表属性可取值的集合，属性 A 的域用`Domain(A)`表示。

### 关系规则

1. 第一范式规则：在定义的表中，关系模型坚持不允许含有多值属性（重复字段）和含有内部结构（列的任何值必须是简单类型）；

2. 只能基于内容存取行（参考 py 字典的无序性）；

3. 行唯一性（任何元组必须唯一）。
